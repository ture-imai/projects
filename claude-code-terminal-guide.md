確かに最初は難しく感じますが、**実際に使う操作はほぼ3つだけ**です：

```
git pull   → GitHubの変更をPCに取り込む
git add    → 変更ファイルを準備する
git commit → 変更を記録する
git push   → PCの変更をGitHubに上げる
```

今日やったことを整理すると：

- **Web画面でファイル作成** → GitHub上に保存
- **git pull** → PCに反映

この流れが体験できたので、あとは繰り返すだけで慣れてきます。`git-usage.md` というファイルも既に持っていらっしゃるので、そこにメモしておくと後から見返せて便利ですよ！
まとめると：

```
GitHub側で編集した
        ↓
  git pull（手動）
        ↓
  ローカルに反映


ローカルで編集した
        ↓
  git add
  git commit
  git push（手動）
        ↓
  GitHubに反映
```

自動同期はしないので、**どちらの方向も必ず手動操作が必要**、という認識で合っています。

ちなみに補足として：
- **どちらかを忘れると差分が生まれてズレる**ので、編集前後に pull/push する習慣をつけると安全です
- 複数人で使う場合は特に `git pull` を忘れずに、が鉄則です
